Thinking like a programmer

1.Procedural Thinking
This means creating perfectly clear, and you should make unambiguous instructions for a computer to follow.
2.Abstract Thinking
Finding similarity and generality among seemingly different things.
3.Systematic Thinking
This happens when you break a big problem down into smaller pieces.
4.Technological Thinkng
Understanding what a computer is, how it works, and what it's good and bad at doing.
5.Systems Thinking
Collect evidence -> Generate theories -> Test those theories -> Fix the problem

inline / block

Inline is handled as a part of text, so it doesn't generate a new paragraph.
That is, a text style.
{br, b, em, img, span, a, strong} tags are all inline.

Block is a block. That's it!
{p, div, form} tags are block.

div VS span

div tag is a block, and this is nomally used.
However, if you want to arrange some part of text, you should use span tag.
